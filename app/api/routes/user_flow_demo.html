Developer Flow

Goal Intake Lab

유저 생성부터 목표 분석, 질문 답변, 계획 생성, 자동 배치, 캘린더 확인까지를 단계별 API 카드로 나눠서 테스트합니다. 입력은 각 단계 옆에 바로 붙여두고, 오른쪽에는 상태와 request/response를 고정해 개발 중 확인하기 쉽게 구성했습니다.

Swagger 열기 API Playground
Step 1

유저 생성

`user_id`를 먼저 확보한 뒤 이후 모든 일정/목표 요청에서 재사용합니다.

POST /api/v1/users
Step 2-3

일정 세팅

고정 일정과 가변 작업을 분리해 테스트할 수 있게 입력 폼도 각 카드로 쪼갰습니다.

고정 일정 등록

반복 일정 여부, 제목, 위치, 설명을 직접 조절할 수 있습니다.

POST /api/v1/schedules/fixed

가변 작업 등록

예상 시간, 세션 길이, 우선순위를 직접 손보며 배치 결과를 보기 좋습니다.

POST /api/v1/tasks/flexible
Step 4-6

목표 분석과 계획 생성

자연어 목표를 분석하고, 생성된 질문은 아래 별도 답변 영역에서 바로 채워 넣습니다.

목표 입력

카테고리를 비워두면 AI가 추론하고, 직접 고정하고 싶으면 선택하세요.

POST /api/v1/goals/intake

계획/캘린더 범위

계획 생성 시 기존 active plan 교체 여부와 배치 범위를 따로 제어합니다.

POST /api/v1/goals/complete · /planner/allocate
질문 답변 입력 후 `목표 저장 + 계획 생성`을 누르고, 그 다음 `배치 + 캘린더 조회`를 실행하세요.
